# Auth 模块

## 模块概述

Auth 模块负责处理用户认证与授权功能，是系统的安全入口。该模块提供用户注册、登录、JWT 令牌管理、登录尝试限制等核心安全功能，确保只有合法用户才能访问系统资源。

## 主要功能

- **用户注册**：新用户账号创建，密码加密存储
- **用户登录**：验证用户凭据，生成 JWT 令牌
- **JWT 认证**：基于令牌的无状态身份验证
- **登录限流**：防止暴力破解，限制登录尝试次数
- **密码加密**：使用安全的哈希算法存储密码

## 模块结构

### controller/ 控制器层

#### `AuthController` 认证控制器
- 用户注册端点：`POST /api/auth/register`
- 用户登录端点：`POST /api/auth/login`
- 获取当前用户信息：`GET /api/auth/me`

### service/ 服务层

#### `AuthService` 认证服务接口
- 定义认证相关的业务方法
- 注册、登录、用户信息获取

#### `AuthServiceImpl` 认证服务实现
- 用户注册逻辑：密码加密、用户创建
- 用户登录逻辑：凭据验证、JWT 生成
- 用户信息获取：从 JWT 解析用户信息

#### `JwtService` JWT 服务
- JWT 令牌生成
- JWT 令牌解析与验证
- 从令牌提取用户信息
- 令牌过期检查

#### `LoginAttemptService` 登录尝试服务
- 记录登录尝试次数
- 检查是否超过限制
- 成功登录后重置计数器
- 防止暴力破解攻击

### dto/ 数据传输对象

#### `RegisterRequest` 注册请求
- 用户名、邮箱、密码
- 新用户注册所需信息

#### `LoginRequest` 登录请求
- 邮箱、密码
- 用户登录凭据

#### `AuthResponse` 认证响应
- JWT 令牌
- 用户信息
- 登录成功后返回

#### `UserDTO` 用户数据传输对象
- 用户基本信息（ID、用户名、邮箱）
- 不包含敏感信息（如密码）

### entity/ 实体层

#### `User` 用户实体
- 用户 ID、用户名、邮箱
- 加密后的密码
- 创建时间、更新时间
- JPA 实体，映射到数据库表

#### `LoginAttempt` 登录尝试实体
- 用户邮箱
- 尝试次数
- 最后尝试时间
- 用于登录限流

### repository/ 数据访问层

#### `UserRepository` 用户仓库
- 基于邮箱查找用户
- 检查用户名是否存在
- 检查邮箱是否存在
- 标准 CRUD 操作

#### `LoginAttemptRepository` 登录尝试仓库
- 基于邮箱查找登录尝试记录
- 标准 CRUD 操作

## 认证流程

### 用户注册流程
1. 客户端发送注册请求（用户名、邮箱、密码）
2. 服务端验证用户名和邮箱是否已存在
3. 对密码进行加密处理（使用 BCrypt）
4. 创建新用户记录
5. 返回成功响应

### 用户登录流程
1. 客户端发送登录请求（邮箱、密码）
2. 服务端检查登录尝试次数（防止暴力破解）
3. 验证用户凭据
4. 生成 JWT 令牌
5. 重置登录尝试计数器
6. 返回 JWT 令牌和用户信息

### JWT 认证流程
1. 客户端在请求头中携带 JWT：`Authorization: Bearer <token>`
2. `JwtAuthenticationFilter` 拦截请求
3. 解析并验证 JWT 令牌
4. 从令牌提取用户信息
5. 将认证信息存入 SecurityContext
6. 允许访问受保护的资源

## 安全特性

### 密码安全
- 使用 BCrypt 算法加密密码
- 每次加密使用随机盐值
- 无法从密文反推原始密码

### 登录限流
- 记录连续失败的登录尝试
- 超过限制后阻止登录
- 成功登录后重置计数器
- 有效防止暴力破解

### JWT 安全
- 令牌包含过期时间
- 使用密钥签名，防止篡改
- 无状态设计，易于扩展
- 支持令牌刷新机制

## 配置说明

### JWT 配置
- 密钥：用于签名和验证令牌
- 过期时间：令牌有效期（默认 24 小时）
- 签发者：令牌签发者标识

### Rate Limit 配置
- 最大尝试次数：连续失败登录的最大次数
- 锁定时间：超过限制后的锁定时长
