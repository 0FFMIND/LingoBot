# MCP 模块

## 模块概述

MCP (Model Context Protocol) 模块是聊天机器人的工具扩展系统，允许 AI 模型通过调用各种工具来获取外部信息或执行特定操作。该模块提供了一个统一的工具注册、管理和执行框架，支持多种类型的工具，并能与不同的 AI 模型无缝集成。

## 主要功能

- **工具注册与发现**：提供统一的工具注册机制，支持动态添加和移除工具
- **工具分类管理**：按使用场景对工具进行分类（一次性工具、仅 Agent 模式、全部模式）
- **工具执行引擎**：负责接收工具调用请求并执行相应的工具
- **格式转换**：提供 MCP 工具定义与 OpenAI 格式之间的转换，支持多种 AI 模型
- **API 接口**：提供 REST API 供外部调用工具和获取工具列表

## 模块结构

### controller/ 控制器层
- `McpController`：提供 MCP 工具相关的 REST API 接口
  - 获取所有工具列表
  - 获取 OpenAI 格式的工具列表
  - 执行单个工具调用
  - 批量执行工具调用

### service/ 服务层
- `McpService`：MCP 服务的核心业务逻辑
  - 工具列表管理
  - 格式转换（MCP ↔ OpenAI）
  - 工具调用转发

- `McpToolRegistry`：工具注册表
  - 工具注册与注销
  - 按模式筛选工具
  - 工具查找与执行

- `McpToolHandler`：工具处理器接口
  - 定义工具的基本规范
  - 所有工具必须实现此接口

- `ToolCategory`：工具分类枚举
  - `ONE_TIME`：一次性工具，普通聊天模式可用
  - `AGENT_ONLY`：仅 Agent 模式可用
  - `ALL`：所有模式可用

### tools/ 工具实现层
- `ListFilesTool`：列出目录文件工具
  - 支持递归遍历
  - 支持 Glob 模式过滤
  - 返回相对路径列表

- `ReadFileTool`：读取文件工具
  - 支持指定起始行和行数限制
  - 自动处理文件大小限制
  - 返回文件内容文本

- `SearchCodeTool`：代码搜索工具
  - 支持正则表达式搜索
  - 支持 Glob 模式过滤文件
  - 返回匹配行及上下文

- `BirdSoundTool`：鸟叫声工具
  - 提供多种鸟叫声类型
  - 支持根据描述自动选择
  - 返回音频 URL 供前端播放

### dto/ 数据传输对象
- `McpTool`：MCP 工具定义
  - 工具名称、描述
  - 参数定义（类型、属性、必需项）

- `McpToolCall`：工具调用请求
  - 调用 ID、工具名称
  - 参数字典
  - 关联对话 ID

- `McpToolResult`：工具执行结果
  - 执行成功标志
  - 结果内容列表
  - 错误信息

### config/ 配置层
- `McpToolConfig`：MCP 工具配置
  - 内置工具的自动注册
  - 服务启动时初始化

## 工具分类说明

### 一次性工具 (ONE_TIME)
适用于普通聊天模式，用户可以直接请求使用。
- 示例：`bird_sound`（鸟叫声工具）

### 仅 Agent 模式 (AGENT_ONLY)
仅在 Agent 模式下可用，用于需要多轮思考和规划的复杂任务。
- 示例：`list_files`、`read_file`、`search_code`（文件操作工具）

### 全部模式 (ALL)
所有模式都可用，通用型工具。

## 使用方式

### 1. 创建新工具
1. 实现 `McpToolHandler` 接口
2. 定义工具元数据（名称、描述、参数）
3. 实现工具执行逻辑
4. 通过 `McpToolRegistry` 注册工具

### 2. 调用工具
- 通过 API 直接调用：`POST /api/mcp/tools/call`
- 通过聊天服务间接调用：AI 模型根据用户请求自动调用

### 3. 模式选择
- 普通聊天模式：仅使用 `ONE_TIME` 和 `ALL` 分类的工具
- Agent 模式：使用所有分类的工具，支持多轮工具调用
