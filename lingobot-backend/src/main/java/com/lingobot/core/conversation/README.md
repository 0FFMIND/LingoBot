# Conversation 模块

## 模块概述

Conversation 模块负责管理用户对话和消息数据，是聊天机器人的数据持久化层。该模块提供对话的创建、查询、更新、删除等功能，并管理消息的存储和检索，为 Chat 模块提供数据支持。

## 主要功能

- **对话管理**：创建、查询、更新、删除对话
- **消息管理**：添加、查询、删除消息
- **用户对话列表**：获取用户的所有对话
- **消息历史**：构建对话消息列表
- **批量操作**：批量删除消息、清理对话

## 模块结构

### controller/ 控制器层

#### `ConversationController` 对话控制器
- 获取用户对话列表：`GET /api/conversations`
- 创建新对话：`POST /api/conversations`
- 更新对话标题：`PUT /api/conversations/{id}`
- 删除对话：`DELETE /api/conversations/{id}`
- 获取对话消息：`GET /api/conversations/{id}/messages`

### service/ 服务层

#### `ConversationService` 对话服务接口
- 定义对话相关的业务方法

#### `ConversationServiceImpl` 对话服务实现
- 创建新对话
- 获取用户的所有对话
- 更新对话标题
- 删除对话
- 添加用户消息和助手消息
- 添加带音频的用户消息
- 获取对话实体
- 从指定索引删除消息

### dto/ 数据传输对象

#### `ConversationDTO` 对话数据传输对象
- 对话 ID、标题
- 创建时间、更新时间
- 最后消息预览

#### `CreateConversationRequest` 创建对话请求
- 对话标题（可选）
- 初始消息内容（可选）

#### `MessageDTO` 消息数据传输对象
- 消息 ID、对话 ID
- 角色（user/assistant）
- 内容
- 音频数据、音频格式、音频时长
- 创建时间

### entity/ 实体层

#### `Conversation` 对话实体
- 对话 ID、用户 ID
- 标题
- 创建时间、更新时间
- 消息列表（一对多关系）
- JPA 实体，映射到数据库表

#### `Message` 消息实体
- 消息 ID、对话 ID
- 角色（user/assistant）
- 内容
- 音频数据、音频格式、音频时长
- 创建时间
- 对话引用（多对一关系）
- JPA 实体，映射到数据库表

### repository/ 数据访问层

#### `ConversationRepository` 对话仓库
- 按用户 ID 查找对话
- 按用户 ID 和对话 ID 查找
- 标准 CRUD 操作

#### `MessageRepository` 消息仓库
- 按对话 ID 查找消息（按时间排序）
- 按对话 ID 批量删除
- 标准 CRUD 操作

## 对话生命周期

### 创建对话
1. 用户请求创建新对话
2. 服务端创建 `Conversation` 实体
3. 设置初始标题（默认或用户指定）
4. 保存到数据库
5. 如果有初始消息，添加第一条消息
6. 返回对话信息

### 发送消息
1. 用户发送消息到指定对话
2. 服务端创建 `Message` 实体
3. 设置角色为 "user"
4. 关联到对应对话
5. 保存到数据库
6. AI 生成响应后，创建 "assistant" 角色的消息

### 获取对话列表
1. 用户请求获取自己的对话列表
2. 服务端按用户 ID 查询
3. 按更新时间倒序排列
4. 转换为 DTO 格式返回

### 删除对话
1. 用户请求删除指定对话
2. 服务端验证对话归属
3. 删除对话及其所有消息
4. 返回成功响应

## 消息管理

### 消息角色
- **user**：用户发送的消息
- **assistant**：AI 助手生成的消息
- **tool**：工具执行结果（临时，不存储）

### 消息类型
- **文本消息**：普通文本内容
- **音频消息**：包含音频数据的消息
  - `audioData`：Base64 编码的音频数据
  - `audioFormat`：音频格式（wav、mp3 等）
  - `audioDuration`：音频时长（秒）

### 消息列表构建
- 按创建时间升序排列
- 支持截断到指定索引
- 转换为 LLM 模型可识别的格式

## 对话标题

### 自动生成标题
- 默认使用第一条消息的前若干字符
- 用户可以后续修改

### 更新标题
- 用户可以在任意时间更新对话标题
- 标题用于在对话列表中显示

## 批量操作

### 批量删除消息
- 从指定索引开始删除所有后续消息
- 用于重试和编辑消息功能
- 保持对话的一致性

### 清理对话
- 删除对话时级联删除所有消息
- 使用数据库外键约束
- 确保数据完整性

## 依赖关系

Conversation 模块被以下模块依赖：
- **Chat 模块**：使用对话和消息存储功能
- **Auth 模块**：用户信息用于对话归属验证
